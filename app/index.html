History & Applications

Origins — The Königsberg Bridges (1736)

Graph theory originated when Leonhard Euler solved the famous Königsberg Bridge Problem. The city had 4 land masses connected by 7 bridges. Euler asked: Can one walk through the city crossing each bridge exactly once? He represented lands as vertices and bridges as edges, proving the answer was no — since more than 2 vertices had an odd degree.

Key Milestones

  • 1736 — Euler: Foundation of graph theory with the Königsberg problem.
  • 19th century — Kirchhoff: Used graphs to analyze electrical circuits.
  • 19th century — Cayley: Applied graphs to chemical molecular structures.
  • 1936 — Kőnig: Published Theory of Finite and Infinite Graphs, formalizing many concepts.
  • 20th century — Erdős & Berge: Advanced coloring, connectivity, and combinatorial optimization.

Applications

Graphs are used in computer science (networks, compilers, web search), chemistry (molecular modeling), biology (DNA sequencing), social sciences (social networks), transportation (routing, GPS), image processing, and AI.

Fundamentals

What is a Graph?

A finite graph G = (V, E) is defined by a finite set V = {v₁, v₂, …, vₙ} of vertices (nodes) and a finite set E = {e₁, e₂, …, eₘ} of edges.

Graph Order & Size

Order of G = |V| = n (number of vertices). Size of G = |E| = m (number of edges).

Vertex Degree

The degree of a vertex is the number of edges incident to it, denoted deg(v).

  • Undirected graph: deg(v) = number of edges touching v.
  • Directed graph: deg(v) = deg⁺(v) + deg⁻(v) where deg⁺ is out-degree and deg⁻ is in-degree.

Key Terms

  • Adjacent: Two vertices connected by an edge.
  • Incident: An edge is incident to the vertices it connects.
  • Predecessor/Successor: In a digraph, if (u,v) ∈ E, then u is a predecessor of v and v is a successor of u.

Directed vs. Undirected Graphs

Undirected Graphs

Edges have no orientation. The edge {u, v} is identical to {v, u}. The relationship is symmetric.

Directed Graphs (Digraphs)

Each edge is an ordered pair (u, v) called an arc, going from source u to target v. Each arc has an origin (source) and a goal (target).

In-Degree & Out-Degree

In-degree deg⁻(v): number of arcs pointing into v. Out-degree deg⁺(v): number of arcs leaving v. For any digraph: ∑deg⁺(v) = ∑deg⁻(v) = |E|.

When to Use Each

  • Undirected: Friendships, two-way roads, chemical bonds.
  • Directed: Web links, one-way streets, dependencies (DAGs), Twitter follows.

Graph Types

Simple Graph vs Multigraph

A simple graph has no loops and at most one edge between any two vertices. A multigraph may have loops and/or multiple edges between the same pair of vertices.

Planar vs Non-Planar

A planar graph can be drawn on a flat surface without any edges crossing. A non-planar graph cannot avoid edge crossings no matter how it is redrawn.

Special Directed Graph Types

  • Symmetric: (x,y) ∈ E ⇒ (y,x) ∈ E — every arc has its reverse.
  • Antisymmetric: (x,y) ∈ E ⇒ (y,x) ∉ E — no arc has its reverse.
  • Transitive: (x,y) ∈ E and (y,z) ∈ E ⇒ (x,z) ∈ E.
  • Regular: All vertices have the same degree.
  • Tournament: A complete directed graph where every pair has exactly one arc (no bidirectional, no missing edges).

Subgraphs

  • Induced subgraph: Remove vertices and all their incident edges.
  • Spanning subgraph: Remove only edges (keep all vertices).

Graph Representations

Adjacency Matrix

An n × n matrix where entry A[i][j] = 1 if there is an edge from vertex i to vertex j, and 0 otherwise.

  • Undirected graph: The matrix is symmetric (A[i][j] = A[j][i]).
  • Directed graph: The matrix is generally non-symmetric. A[i][j] = 1 means an outgoing arc from i to j.

Arc Incidence Matrix

An n × m matrix linking vertices to edges.

  • Undirected: A[i][j] = 1 if vertex i is incident to edge j, 0 otherwise.
  • Directed: A[i][j] = +1 if vertex i is the source of arc j, −1 if target, 0 otherwise.

Paths, Chains, Cycles & Circuits

Chain (Undirected) vs Path (Directed)

A chain (in an undirected graph) is a sequence of vertices connected by edges. A path (in a directed graph) follows arc directions. Length = number of edges used.

Elementary vs Simple

  • Elementary chain/path: Cannot visit the same vertex twice.
  • Simple chain/path: Cannot use the same edge twice (but may revisit vertices).

Cycle vs Circuit

  • Cycle: A simple closed chain (undirected) — starts and ends at the same vertex, no repeated edges.
  • Circuit: A simple closed path (directed) — follows arc directions back to start.

Acyclic Graphs

A graph with no cycles is acyclic. A connected acyclic undirected graph is a tree. A directed acyclic graph is a DAG — crucial for topological sorting and dependency resolution.

Connectivity & Strongly Connected Components

Connected Graph

An undirected graph is connected if there exists a chain between any pair of vertices. For a directed graph, we speak of connectivity if, forgetting edge orientation, the graph is connected.

Strongly Connected Graph

A directed graph is strongly connected if there is a directed path from any vertex to any other vertex.

Marking Algorithm for SCCs

  1. Select a vertex, mark it with +.
  2. Mark all successors (and successors of successors) with +.
  3. Mark all predecessors (and predecessors of predecessors) with −.
  4. Vertices marked with both + and − form an SCC.
  5. Remove the SCC and repeat on remaining vertices.

Example

Eulerian & Hamiltonian Problems

Eulerian Problem

An Eulerian chain/cycle uses each edge exactly once. A graph is Eulerian if it admits an Eulerian cycle.

Euler's Theorem

  • If more than 2 vertices have an odd degree → not Eulerian.
  • If exactly 2 vertices have an odd degree → Eulerian path exists (start/end at those vertices).
  • If all vertices have even degree → Eulerian cycle exists (start from any vertex).

Hamiltonian Problem

A Hamiltonian path visits each vertex exactly once. A Hamiltonian cycle visits every vertex exactly once and returns to start. A graph admitting a Hamiltonian cycle is Hamiltonian.

NP-Complete

The Hamiltonian cycle problem is NP-complete — no known efficient algorithm solves it for all graphs. This is related to the Traveling Salesman Problem.

Trees, Forests & Spanning Trees

Bridge

An edge is a bridge (isthmus) if its removal disconnects the graph — i.e., it increases the number of connected components. Equivalently, a bridge is an edge that does not belong to any cycle.

Tree — Five Equivalent Properties

A tree is a connected graph containing no cycle. All edges of a tree are bridges. For a graph G with n vertices:

  1. P1: G is connected and acyclic.
  2. P2: G is connected and minimal (removing any edge disconnects it).
  3. P3: G is connected and has exactly n − 1 edges.
  4. P4: G is acyclic and maximal (adding any edge creates a cycle).
  5. P5: There is a unique path between any two vertices.

Special Tree Structures

  • Spanning tree: A maximum subgraph that is a tree (contains all vertices, no cycles).
  • Rooted tree: A tree with a distinguished root vertex — unique path from root to every vertex.
  • Forest: A set of trees (acyclic graph, possibly disconnected).

Minimum Spanning Tree Algorithms

The MST Problem

Given a connected, weighted graph, find a spanning tree that minimizes the total edge weight. For a Maximum Spanning Tree, maximize instead.

Kruskal's Algorithm (1956)

Steps

  1. Sort all edges by weight in ascending order.
  2. Add edges one by one to the tree, skipping any edge that would create a cycle.
  3. Stop when the tree has n − 1 edges (covers all vertices).

Kruskal maintains an acyclic forest, merging components.

Prim's Algorithm (1957)

Steps

  1. Choose any starting vertex.
  2. From all edges connecting the tree to non-tree vertices, pick the minimum-weight edge.
  3. Add the edge and its incident vertex to the tree.
  4. Repeat until all vertices are in the tree.

Prim grows a single tree, one vertex at a time.

Key Difference

Kruskal sorts globally and merges forests. Prim grows locally from a single vertex. Both produce an MST with the same total weight (though the tree structure may differ).

Graph Coloring, Bipartite & Line Graphs

k-Coloring & Chromatic Number

A graph admits a k-coloring if its vertices can be partitioned into k independent sets such that no two adjacent vertices share the same color. The chromatic number γ(G) is the minimum k for which G is k-colorable.

Four Color Theorem

Every planar graph is 4-colorable. No matter how complex the map, 4 colors always suffice to color regions so that adjacent regions have different colors.

Bipartite Graphs

A graph is bipartite if its vertices can be partitioned into two disjoint sets U and W such that every edge connects a vertex in U to one in W. Equivalently: γ(G) = 2, or G contains no odd-length cycles.

Line Graphs L(G)

The line graph L(G) represents edges of G as vertices: two vertices in L(G) are adjacent if the corresponding edges in G share a common endpoint. Edge coloring of G ≡ vertex coloring of L(G).

Network Flows & Ford-Fulkerson

Flow Networks

A flow network is a directed, weighted graph with a source S (only outgoing edges) and a sink T (only incoming edges). Each arc has a capacity c (max flow allowed) and an actual flow f.

Flow Conservation Law

For every vertex except S and T: ∑ f(in) = ∑ f(out). The sum of flows arriving equals the sum leaving.

Key Definitions

  • Saturated arc: flow = capacity (direct) or flow = 0 (reverse).
  • Augmenting path: A path from S to T where flow can still be increased.
  • Bottleneck: min(capacity − flow) along an augmenting path.

Ford-Fulkerson Algorithm

Steps

  1. Initialize all flows to 0. Set max_flow = 0.
  2. Find an augmenting path from S to T using BFS (Edmonds-Karp variant).
  3. Calculate bottleneck = min(capacity − flow) along the path.
  4. Update flows: add bottleneck to forward arcs, subtract from reverse arcs.
  5. max_flow += bottleneck. Repeat from step 2.
  6. Stop when no augmenting path exists. max_flow is the answer.

Worked Example

Given the network below with capacities on arcs:

  • Iteration 1: Path S→c→b→T, bottleneck = min(8,6,4) = 4. Flow = 4.
  • Iteration 2: Path S→c→d→T, bottleneck = min(4,3,9) = 3. Flow = 7.
  • Iteration 3: Path S→a→d→T, bottleneck = min(3,2,6) = 2. Flow = 9.
  • No more augmenting paths → Maximum flow = 9.

Cyclomatic & CoCyclomatic Numbers

  • Cyclomatic number: μ(G) = m − n + p — number of independent cycles. (m=edges, n=vertices, p=connected components)
  • CoCyclomatic number: λ(G) = n − p — number of independent cut sets.
